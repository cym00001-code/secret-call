# 全局要求

生效日期：2026-05-22

本文件记录本项目后续开发、部署和发布必须长期遵守的全局要求。除非用户明确修改，本要求优先作为后续版本规划与执行依据。

## 工程化目录要求

仓库目录职责以 `PROJECT_STRUCTURE.md` 为准。后续开发必须遵守：

- 生产 Web、服务端、移动端、下载页、部署配置、参考原型分目录维护。
- Kimi 或其他外部工具生成的前端只能进入 `references/` 作为参考，不直接接入生产构建、发布、安全协议或状态机。
- 下载页源文件放在 `download/`，APK 等二进制发版产物不提交 Git，只上传服务器。
- 临时交接文档、测试截图、构建产物、日志和本地数据不得散落在根目录。

## 产品升级方向

当前项目将从网页密聊逐步升级为包含移动端 App 的安全通信产品。

核心目标：

- 保留现有网页端安全会话能力，并继续维护公网可访问版本。
- 新增 Android App。
- 新增 iOS App。
- 新增服务器安装包下载网页，用于托管并分发最新版安装包。

## 截屏与截图防护

Android 端：

- 必须支持系统级阻止截屏和录屏。
- 在敏感会话页面启用系统安全窗口能力，例如 Android `FLAG_SECURE`。
- 检测到相关风险或防护状态变化时，需要在会话内提醒。

iOS 端：

- iOS 系统不允许第三方 App 真正阻止系统截图。
- iOS 端目标是检测截图行为，并在会话内自动提醒双方。
- 可使用系统截图通知能力，例如 `UIApplicationUserDidTakeScreenshotNotification`。
- 如涉及录屏，应尽量检测屏幕捕获状态并提醒，但不能承诺系统级彻底阻止。

通用要求：

- 不得在产品文案或技术方案中承诺无法实现的系统级能力。
- 必须清楚区分“阻止截屏”“检测截图”“提醒会话成员”三类能力。
- 任一端检测到截图或录屏风险后，会话内必须产生明确提示。

## 安装包托管网页

需要在当前云服务器上新增一个网页，用于托管 App 安装包。

要求：

- 用户打开网页后可以下载最新版安装包。
- Android APK 必须提供明确下载入口。
- iOS 若使用 TestFlight、企业签名、描述文件或其他分发方式，也必须在该网页说明当前可用入口。
- 页面必须展示当前版本号、更新时间、平台、安装包大小和更新说明。
- 每完成一版发布，下载页必须同步更新到最新状态。

建议服务器路径：

- 下载页目录：`/www/wwwroot/secret-room-downloads`
- Android 安装包目录：`/www/wwwroot/secret-room-downloads/releases/android`
- 版本清单：`/www/wwwroot/secret-room-downloads/releases/manifest.json`

实际路径以后可根据服务器 Nginx 配置调整，但必须保持文档同步。

## 每版发布工作流

从现在开始，每完成一个可用版本，必须按以下顺序发布：

1. 本地完成开发与验证。
2. 同步到 Git 仓库。
3. 构建移动端安装包。
4. 上传 APK 或其他安装包到云服务器。
5. 更新服务器安装包下载网页。
6. 部署或更新网页端服务。
7. 做公网冒烟测试。
8. 向用户汇报 Git 状态、服务器下载地址、安装包版本和验证结果。

除非用户明确要求跳过，否则不要只本地修改而不说明未发布状态。

固定 Android 发布入口：

```powershell
.\scripts\release-android.ps1
```

脚本职责：

- 构建网页静态资源并同步到 Capacitor。
- 构建已签名 Android release APK。
- 校验 APK 签名、zip 对齐、大小和 SHA256。
- 更新 `download/releases/manifest.json`。
- 上传下载页、图标、manifest 和 APK 到服务器下载目录。
- 下载公网 APK 并校验 SHA256。

签名要求：

- 签名文件和密码只放在 `apps/mobile/android/local-signing/`。
- `local-signing/` 不提交 Git。
- 只提交 `release.properties.example` 作为模板。
- 发布脚本不得把签名密码写入日志或提交文件。

## Git 要求

每个可用版本都必须同步到 Git 仓库。

要求：

- 提交前检查工作区，避免混入无关文件。
- 提交信息要能说明本版主要变化。
- 推送后向用户说明分支名、提交号和是否已创建 Pull Request。
- 如果 GitHub CLI 未登录或远端推送失败，必须明确说明阻塞点。

## 服务器发布要求

服务器信息以 `C:\Users\Administrator\OneDrive\Desktop\编程项目\服务器说明.md` 为准。

当前密聊项目服务器路径：

- 项目目录：`/www/wwwroot/secret-room`
- PM2 服务：`secret-room-server`、`secret-room-web`
- 公网地址：`https://8.138.150.200/`

发布要求：

- 构建成功后再重启服务。
- 重启后检查 PM2 服务状态。
- 检查公网页面可访问。
- 涉及下载页时，检查安装包 URL 可访问。
- 涉及缓存问题时，必须确认 Nginx 或浏览器不会拿到旧资源。

## 验证要求

每次发布至少验证：

- 网页端首屏可打开。
- 双端进入房间正常。
- 消息发送、查看、焚毁逻辑没有回退。
- 如果本版涉及图片/视频，必须验证点击查看前不暴露内容。
- 如果本版涉及 App，必须验证安装包可下载，并记录版本号。
- 如果本版涉及截屏检测或防护，必须分别说明 Android 和 iOS 的实际验证结果。

## 重要限制

- Android 可以做系统级截屏/录屏阻止。
- iOS 只能检测截图并提醒，不能真正阻止用户使用系统截图。
- 拍照、外部摄像头、另一台设备录制等物理旁路无法被软件彻底阻止。
- 任何安全能力都必须如实描述，不得夸大。
